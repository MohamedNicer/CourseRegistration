# Contributing to Course Registration System

Thank you for your interest in contributing to this project!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Submit a pull request

## Development Setup

See [README.md](README.md) for installation and setup instructions.

## Code Style

- Follow existing code patterns
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Testing

Before submitting a pull request:

1. Test all three portals (Student, Instructor, Admin)
2. Verify authentication flows work correctly
3. Check that data filtering works as expected
4. Test CRUD operations in Admin portal
5. Verify analytics charts display correctly

## Pull Request Guidelines

- Provide a clear description of the changes
- Reference any related issues
- Include screenshots for UI changes
- Ensure all tests pass
- Update documentation if needed

## Questions?

Feel free to open an issue for any questions or concerns.
